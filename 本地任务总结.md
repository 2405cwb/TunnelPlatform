# 本地任务总结

## 1. 当前目标

本项目是一个本地化的地铁隧道检测成果上传与展示平台，核心目标是：

- 用 WinForms 工具同步台账、上传站点/区间成果数据。
- 用 .NET WebAPI 解析二维/三维 SQLite、保存图像和点云文件、写入 PostgreSQL。
- 用前端展示平台按工程实例、站点/区间、里程、病害类型展示二维图、病害、环片和点云断面。
- 给后续 C 端或其他前端同事提供清晰 API 调用方式。

工程实例的定义是：

```text
线路名称 + 上下行 + 采集日期
```

这些信息全部来自 `台账.xlsx`，不再让用户手工输入。

## 2. 当前技术栈

- 上传端：`.NET 10 WinForms`
- 服务端：`.NET 10 WebAPI`
- 数据库：PostgreSQL，默认库名 `tunnel_platform`
- ORM：Entity Framework Core + Npgsql
- 日志：Serilog
- SQLite 解析：`Microsoft.Data.Sqlite`
- Excel 台账读取：`ClosedXML`
- 展示前端：原生 `HTML + CSS + JavaScript`，由 API 的 `wwwroot` 直接托管

默认访问地址：

```text
展示平台：http://localhost:5140/
Swagger：http://localhost:5140/swagger
```

## 4. 当前数据库方向

项目已经从早期临时业务表，调整为以 Word 6.2 数据库设计中的成果表为主体。

当前保留的平台上下文表：

- `projects`：线路/逻辑工程。
- `project_instances`：线路 + 上下行 + 采集日期。
- `entity_datasets`：某工程实例下某站点/区间的一次上传批次。

当前作为主成果表的 Word 增强表：

- `STATION`
- `RING_LOC`
- `RING_FIT`
- `HORSE_DIST`
- `INTER_RING_PLATFORM`
- `INNER_RING_PLATFORM`
- `LIMIT_CHK_DATA`
- `DISEASE_CHK_DATA`
- `SEC_PTCLOUD_TATA`
- `SEC_PTCLOUD_DATA` 视图
- `IMAGE_DATA`

所有成果表统一增加了平台上下文字段，例如：

- `RowGuid`
- `DatasetId`
- `ProjectInstanceId`
- `StationID`
- `SourceCategory`
- `SourceTable`
- `CreatedAt`
- `RawJson`

这样既满足“必须以 Word 设计表为基础”的工作要求，也能支撑多线路、多上下行、多采集日期在一个 PostgreSQL 中统一管理。

## 5. 数据来源与二维优先原则

当前已落实的关键原则：

- 只要存在 `01二维数据`，病害优先写入 2D `BASIC_DISEASE` 到 `DISEASE_CHK_DATA`。
- 只要存在 `01二维数据`，环片位置优先写入 2D `RING_TUNNEL` 到 `RING_LOC`。
- 没有 2D 时，才使用 3D 病害表和 3D `CIRCLED_FLAKE` 补充。
- 3D 的结构成果仍写入对应 Word 表，例如 `FIT_DIAMETER -> RING_FIT`、`HORSE_DIST -> HORSE_DIST`、`LIMIT_DATA -> LIMIT_CHK_DATA`、`PLAT_FORM -> INTER_RING_PLATFORM`。

二维 SQLite 中的中文编码已按 `GB18030` 兼容读取，后续重新导入时可以避免中文乱码。

## 6. 已实现能力

WinForms 上传工具：

- 选择 API 地址、台账和用户上传数据目录。
- 同步台账并自动生成工程实例。
- 按工程实例展示站点/区间列表。
- 上传勾选实体或上传全部可用实体。
- 删除工程实例、删除站点/区间。
- 查看服务器文件树。
- 显示进度和日志。

WebAPI：

- 台账同步。
- 单站点/区间上传。
- 解析二维/三维 SQLite。
- 保存 `03灰度图`、`04点云`、`05二维病害高清图`。
- 写入 Word 增强版成果表。
- 提供病害分页、工程概览、里程范围、环片、图片和文件树查询接口。
- 托管前端展示平台。

前端展示平台：

- 工程实例选择。
- 站点/区间列表。
- 二维图连续浏览，支持鼠标滚轮切图。
- 环片叠加显示；样例里程不一致时按环序预览。
- 病害列表，筛选后自动刷新，双击打开高清图。
- 病害统计支持当前区间和当前工程两个维度。
- 中间主舞台支持二维图和点云断面 Tab 切换。
- 点云帧按 `04点云` 文件树预留帧号浏览入口。
- 深色/白色主题切换。

## 8. 当前注意事项

- Swagger 默认示例 GUID 不是实际工程实例 ID，调用接口时要先通过 `GET /api/query/project-instances` 获取真实 `projectId`。
- 样例数据中灰度图里程和环片里程没有按真实数据填写，所以前端会启用环序预览；正式数据里程对齐后会按真实里程叠加。
- 旧导入造成的数据库中文乱码无法可靠反推，建议删除对应站点/区间后重新上传。
- `04点云` 当前样例为空，前端已按未来帧号文件浏览模式预留。
